Azure Blob Storage #

Monitor Specs #

Description

Tests the Azure Blob Storage service to validate that blobs can be added, deleted, and retrieved, and that containers and storage accounts can be created.

Name

azureblob

Publisher

Metrist

Version

0.1.0-beta

¬†

Pre-Requisites #

  1. Create your Metrist account. Either get an invitation from someone in your team or create an individual account.

  2. Install Metrist Orchestrator in your own environment.

  3. Configure Orchestrator with your Metrist API Token and environment variables relevant to this monitor.

  4. Register the monitor using Metrist Monitor API.

  5. Observe the monitor in your Metrist.io account.

Orchestrator Environment #

In Orchestrator (the instance in which you want this monitor to run), ensure you have defined at least these environment variables:

sh
METRIST_API_TOKEN=""
METRIST_INSTANCE_ID=""
METRIST_RUN_GROUPS=""

Monitor Environment #

With a Metrist account and Orchestrator installed, you are ready to configure Orchestrator for use with your new monitor.

sh
# (Required) The domain name associated with users in your Active Directory. Example: metrist.io.
METRIST_DOMAIN=""

# (Required) The client ID is the unique Application (client) ID assigned to your app by Azure AD when the app was registered. You can find the Application (Client) ID in your Azure subscription by Azure AD => Enterprise applications => Application ID.
METRIST_CLIENT_ID=""

# (Required) Secret string previously shared with AAD at application registration to prove the identity of the application (the client) requesting the tokens.
METRIST_CLIENT_SECRET=""

# (Required) Tenant ID of the Azure AD tenant or a domain associated with this Azure AD tenant, in order to sign-in a user of a specific organization only.
METRIST_TENANT_ID=""

INFO

These values are never transmitted to Metrist ‚Äî they are never exposed outside your environment.

IMPORTANT

You will need to restart Orchestrator to be sure the new environment variables are available to your new monitor.

Monitor Config Registration #

With monitor configuration now available in your local instance of Orchestrator, the last step is to inform Metrist of your monitor. Metrist will then run schedule the appropriate tests and record the relevant telemetry data (e.g., ‚Äúorchestrate‚Äù).

In the following code, adjust the values of interval_secs and run_groups to suit your needs. run_groups must include at least 1 value in the list defined in METRIST_RUN_GROUPS environment variable. Leave all other values as they are defined below.

json
{
  "monitor_logical_name": "azureblob",
  "interval_secs": 120,
  "run_groups": [],
  "run_spec": {
    "name": "azureblob",
    "run_type": "dll"
  },
  "steps": [{
    "check_logical_name": "CreateStorageAccount",
    "description": "Creates a storage account.",
    "timeout_secs": 900
  }, {
    "check_logical_name": "CreateContainer",
    "description": "Creates a container.",
    "timeout_secs": 900
  }, {
    "check_logical_name": "CreateBlob",
    "description": "Creates a blob in the container.",
    "timeout_secs": 900
  }, {
    "check_logical_name": "GetBlob",
    "description": "Gets the blob from the container.",
    "timeout_secs": 900
  }, {
    "check_logical_name": "DeleteBlob",
    "description": "Deletes the blob from the container.",
    "timeout_secs": 900
  }]
}

Convert your monitor config to a JSON string, get your Metrist API token, and use the curl request below to register your monitor:

sh
json= the json above converted to string

echo $json

api_token=YOUR_TOKEN

echo $api_token

curl -d $json -H "Content-Type: application/json" -H "Authorization: Bearer $api_token" 'https://app.metrist.io/api/v0/monitor-config'

TIP

Tip: Our API documentation describes how to delete/unregister the monitor.

That HTTP request should return a key such as 11zLBF1eZq9g3wZh86nYXV8. And, if you were running Orchestrator in a terminal window, you will see the following output with a recent timestamp:

sh

Metrist now runs the monitor through your Orchestrator. As soon as your Orchestrator reports telemetry data to Metrist, your new monitor will be visible at app.metrist.io.
